    给家人部署一只 AI 助手（OpenClaw 多 Agent 实战）

    最后更新：2026-03-09 部署人：makino 用途：在已有 VPS 上给老婆加一只飞书 AI 机器人，共享服务器、独立灵魂

    背景

    我已经有一只 OpenClaw 飞书机器人（代号「马仔」），跑在海外 VPS 上，日常帮我搜索、写文档、管日程。老婆看了说也想要一只。

    她的需求很简单：日常问答、记东西、查天气、操作飞书日历和文档。不需要写代码，不需要投资分析，不需要 Obsidian 同步——就是一个贴心的生活助手。

    于是我决定在同一台 VPS 上，给她也养一只。

    整体架构

    同一台 VPS（1C2G 新加坡）
├── OpenClaw Gateway（统一网关）
│   ├── 马仔（我的 agent）    ← 飞书应用 A
│   └── 小六（老婆的 agent）  ← 飞书应用 B（独立应用）
├── 共享 Skills（天气、搜索、读网页……）
└── 各自独立的 Workspace（性格、记忆、用户画像）

    关键设计：两只龙虾共用一台服务器、一个 Gateway 进程，但有各自独立的「大脑空间」（workspace）。它们互相看不到对方的记忆和配置。

    第一步：选模型（AI 的大脑）

    这是讨论最久的部分。OpenClaw 通过 API 调用大语言模型，所以需要一个 API 来源。

    我们考虑了这些方案：

    方案 说明 结果
    ChatGPT Plus / Teams 订阅 这是网页聊天的订阅，跟 API 完全是两套东西 ❌ 不适用
    OpenAI API 直充 官方 API，但只支持 Visa/Mastercard ❌ 没有外币卡
    国内大模型 API Deepseek、硅基流动等，支持支付宝 ❌ 被否决（质量不够）
    OpenRouter 第三方中转平台，支持支付宝，可调 GPT/Claude ✅ 备选
    海鲜市场 + OAuth 买 ChatGPT Team 共享号，通过 OAuth 接入 ✅ 最终方案

    最终选了「海鲜市场方案」：花十几块买一个 ChatGPT Team 共享号，OpenClaw 支持通过 OAuth 方式接入，不需要 API Key。

    第二步：在 VPS 上创建新 Agent

    SSH 连到 VPS，一条命令创建新 agent：

    openclaw agents add wifey --name "小六"

    这会自动创建独立的工作空间目录，和已有的马仔完全隔离。

    第三步：配置 AI 模型凭证

    需要从 ChatGPT 网页端提取 OAuth Token。

    步骤： 1. 在浏览器里登录 ChatGPT Team 子号 2. 从浏览器的 session 信息中提取 access_token 和 refresh_token 3. 通过 SSH 写入 VPS 的 agent 认证配置文件

    踩坑：自动化浏览器工具（Playwright）和你日常用的 Chrome/Safari 是独立的，不共享登录状态。需要在自动化浏览器里重新登录。

    第四步：创建飞书应用

    这是工作量最大的一步。需要在飞书开发者后台创建一个全新的应用。

    4.1 为什么需要独立应用？

    一开始我想复用马仔的飞书应用，通过识别发消息的人来路由到不同 agent。但飞书的机制是：个人版飞书的自建应用，对方搜不到也加不了。

    所以只能给老婆的飞书单独创建一个应用。

    4.2 创建应用

    1. 用老婆的飞书账号登录 飞书开放平台
    2. 创建自建应用，取个名字（后来改叫「小六」）
    3. 在「添加应用能力」中勾选 机器人
    4. 记下 App ID 和 App Secret

    4.3 配置消息接收方式

    在「事件与回调」页面：

    • 选择 WebSocket 长连接（推荐！不需要配置 webhook URL，不需要公网暴露端口）
    • 添加事件：im.message.receive_v1（接收消息）

    注意：保存订阅方式时，飞书会检测应用是否有活跃的 WebSocket 连接。所以要先去 VPS 配好凭证、重启 Gateway，等连接建立后再回来保存。

    4.4 VPS 侧配置

    SSH 到 VPS，在 openclaw.json 中为新 agent 添加飞书配置：

    {
  "appId": "你的App ID",
  "appSecret": "你的App Secret",
  "botName": "小六",
  "connectionMode": "websocket",
  "dmPolicy": "open",
  "groupPolicy": "mention",
  "agentId": "wifey"
}

    关键配置说明： - connectionMode: websocket — 长连接，VPS 不需要开放额外端口 - dmPolicy: open — 接受所有人的私聊 - groupPolicy: mention — 在群里只有被 @小六 时才回复，不会每条消息都抢答

    4.5 发布第一个版本

    回到飞书开发者后台： - 创建版本 1.0.0 - 可用范围：先设为自己（测试） - 点击发布（自建应用免审核，秒通过）

    重启 Gateway，看到日志里出现 ws client ready 就说明连上了。

    第五步：首次对话与配对

    老婆在飞书里找到小六，发了第一条消息。

    结果收到一条英文提示：

    Access not configured.
Pairing code: XXXXXX
Ask the bot owner to approve with:
openclaw pairing approve feishu XXXXXX

    这是 OpenClaw 的安全机制：新用户第一次聊天需要管理员批准（类似配对）。

    在 VPS 上执行批准命令即可：

    openclaw pairing approve feishu XXXXXX

    之后老婆再发消息就能正常回复了。

    第六步：预制灵魂

    这是最有趣的部分——给 AI 定义性格和认知。

    在 workspace 目录下创建核心文件：

    SOUL.md（性格）

    定义小六怎么说话： - 直接、利索、说重点 - 不废话、不说教、不主动找人 - 可以口语，但不能墨迹 - 没有依据的话要说清楚是猜测

    USER.md（用户画像）

    告诉小六「主人」是谁： - 高校行政人员 - 作息时间（8:30-17:30 上班，晚上陪孩子，22:30 睡觉） - 讨厌废话 - 有个儿子叫当当，2022 年出生 - 老公的各种别名（小六遇到任何一个都要认出来是同一个人）

    IDENTITY.md（身份）

    明确小六是谁： - 名字叫小六 - 是璐璐的私人助手 - 不是马仔、不是小龙虾（防止跟另一只 agent 串台）

    踩坑：刚开始小六自我介绍时说「我是马仔」——串台了！原因是身份认知不够强。修复方法是在 SOUL.md 最顶部加「最高优先级」的身份声明。

    第七步：开通飞书权限（最曲折的部分）

    飞书的权限系统是整个部署中最折腾的环节。

    7.1 两种权限类型

    飞书有两种完全独立的权限：

    类型 用途 场景
    应用身份权限 以「应用」的身份操作 发通知、查通讯录
    用户身份权限 代替「用户」操作 建日程、写文档、创建任务

    这是最大的坑：我第一轮开了 181 个权限，全是「应用身份」的。但日历、文档、任务这些操作需要「用户身份」权限。两个标签页长得一模一样，很容易搞混。

    7.2 权限开通时间线

    轮次 开通内容 数量
    第一轮 应用身份权限（日历/文档/任务/搜索/多维表格） 181 个
    第二轮 用户身份权限（日历/文档/任务/消息/通讯录/搜索） 188 个
    第三轮 群组相关权限 52 个
    单独补充 cardkit:card:write、offline_access 等 5 个

    好消息是：飞书自建应用的权限大部分是「免审即时生效」的，不需要等审核。

    7.3 日历功能修了四层

    让小六建日程，报错 need_user_authorization。这个问题修了四层才彻底解决：

    第一层：缺用户身份权限 → 开通 188 个用户身份权限

    第二层：OAuth 授权卡片发不出去 → 缺少 cardkit:card:write 权限。这个权限控制的是「机器人能否给用户发卡片消息」。没有这个权限，OAuth 授权链接发不到用户手里，整个授权流程静默失败——用户既看不到错误也看不到授权入口。

    第三层：AI 记住了之前的失败 → 修好权限后 AI 还是说"不行"——因为 AI 的会话记忆里记住了之前的失败，不愿意重试。需要重启 Gateway 清掉会话缓存。

    第四层：缺少 offline_access → OAuth Device Flow 需要 offline_access 权限才能获取 refresh_token。

    修好这四层后，用户终于收到了授权卡片，点击授权，日程创建成功。

    7.4 每个新功能首次使用都要授权一次

    飞书把权限分成了若干「scope 组」。每个组第一次使用时，用户会收到一张授权卡片，点一次就永久生效：

    • 日历 → 点一次授权
    • 云文档 → 点一次授权
    • 任务 → 点一次授权
    • 通讯录 → 点一次授权

    授权完之后就顺畅了，不会再弹。

    第八步：配置技能包

    从马仔那边挑了适合生活场景的 Skills 搬过来：

    搬过来的

    Skill 用途 改了什么
    weather 查天气 默认城市改成深圳，加了生活建议（带伞/穿外套）
    summarize 总结文章/链接 不变
    self-improving-agent 自动记录错误 不变

    没搬的

    • 投资相关（股票监控、交易策略、期权分析）— 用不上
    • 运维相关（安全审计、系统诊断）— 用不上
    • 内容创作管线（碎片收集、文章生成）— 用不上

    全局共享的

    这些 Skills 装在 VPS 全局目录，两只龙虾都能用： - 搜索引擎聚合：17 个搜索引擎 - 网页阅读器：读取任意网页内容 - Tavily 搜索：AI 增强搜索（需要 API Key） - 新闻订阅：RSS 新闻 - 股票查询：查行情（虽然老婆不炒股，但万一要查呢）

    配置搜索 API

    给 VPS 加了 Tavily API Key（免费额度 1000 次/月），两只龙虾共享。

    第九步：写操作手册

    创建 AGENTS.md，相当于小六的「员工手册」：

    • 飞书工具路由表：老婆说什么话 → 小六用什么工具
    • "建个日程" → 飞书日历
    • "记一下要做的" → 飞书任务
    • "帮我列个清单" → 飞书文档
    • "查一下xxx" → 通讯录
    • "帮我跟老公说xxx" → 发消息
    • 搜索决策：什么问题用什么搜索方式
    • 记忆管理：什么信息写哪个文件
    • 回复长度控制：简单问题一句话，复杂问题不超过 5 条
    • 做事规矩：接到就做、做完带结果、做不了就说

    第十步：解决跨组织发消息

    老婆让小六「给老公发消息」，小六说找不到人。

    问题根因

    飞书的应用只能看到同一个组织里的人。小六是老婆组织下的应用，老公不在这个组织里，所以查不到。

    解决方案

    让老公加入老婆的飞书组织：

    1. 在飞书管理后台获取组织邀请链接
    2. 老公在飞书里打开链接，加入组织
    3. 加入后通过 API 查到老公的 open_id（每个用户在每个应用下有唯一标识）
    4. 把 open_id 写到 USER.md 里，以后小六可以直接发消息

    额外踩坑

    • 加入 ≠ 激活：老公通过链接加入了组织，但状态是「未激活」，收不到机器人消息。需要在飞书 App 里切换到新组织，完成激活
    • 多账号限制：老公在飞书里有两个身份（个人号 + 老婆组织号），两个账号分开的。马仔在个人号，小六的消息在老婆组织号。飞书会跨账号推送通知，不会漏消息
    • 两只龙虾无法同群：不同组织的应用互不相通。老婆组织的群可以有「老婆 + 老公 + 小六」，但马仔进不来

    第十一步：最终审计

    全面检查所有配置文件、权限、Skills，评分 B+。修了 8 项小问题：

    • 删掉初始化引导文件（已完成初始化）
    • 补充操作手册中的发消息路由
    • 完善 Skills 索引（从 3 个补全到 12 个）
    • 创建缺失的错误记录模板文件
    • 系统提示词加上 TOOLS.md
    • 日历路由加时区提醒（VPS 在新加坡是 UTC，要换算成北京时间）
    • 清理空的 git 仓库

    重启 Gateway，全部生效。

    踩坑总结（给后来人）

    # 坑 教训
    1 飞书有两种权限：应用身份 vs 用户身份 日历/文档/任务需要「用户身份权限」，不是「应用身份权限」。两个标签页长一样，仔细看
    2 OAuth 授权卡片需要 cardkit:card:write 没这个权限，授权链接发不到用户手里，静默失败
    3 Device Flow 需要 offline_access 否则拿不到 refresh_token
    4 AI 会记住失败 修好权限后要重启 Gateway 清缓存，否则 AI 不会重试
    5 可用范围别设「部分成员」 设了之后新成员看不到机器人，也拉不进群
    6 跨组织不互通 飞书应用只能看到同组织的人，想发消息要先加组织
    7 加入组织 ≠ 激活 加入后要在 App 里切换到新组织完成激活
    8 身份串台要治根 多 agent 共存时，在 SOUL.md 里明确「你不是 xxx」
    9 WebSocket 连接是前提 先启动 Gateway 建立连接，再去飞书保存订阅方式
    10 手机号别写反 多人信息核对三遍
